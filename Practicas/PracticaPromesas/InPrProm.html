Práctica de Temporizadores

Ejercicio 1

Crea una pequeña simulación de proceso de compra online.
La página tendrá:
• Un campo de texto para introducir el nombre del producto.
• Otro campo para la dirección de envío.
• Un botón “Realizar compra”.
• Un elemento p para mostrar el resultado.
Cuando el usuario pulse el botón, deben ocurrir dos pasos encadenados:
1. Función procesarPago(producto) que devuelve una promesa.
Espera 1,5 segundos antes de resolverse o rechazarse.
Si se ha introducido un producto, se resuelve con el mensaje " Pago procesado correctamente para [producto]".
Si el campo está vacío, se rechaza con el mensaje " No se puede procesar el pago sin producto".
2. Función enviarPedido() que devuelve una promesa.
Se ejecuta solo si el pago fue correcto.
Valida que la dirección no esté vacía. Si está vacía, se rechaza con "Dirección de envío no válida".
Si es válida, espera 2 segundos y tiene un 20% de probabilidad de error aleatorio:
• Si hay error → reject("Error durante el envío, intente de nuevo.")
• Si no hay error → resolve("Pedido enviado correctamente a [dirección]")
El resultado final (o el error) debe mostrarse en un p del DOM.

Ejercicio 2

Crea una página que muestre una lista de personas (nombre, teléfono y edad). Incluye dos formularios con los datos necesarios y botones:
• “Añadir persona” → llama a la función newPerson() que devuelve una promesa.
Si el teléfono no existe, añade la persona y muestra un mensaje de éxito.
Si el teléfono ya existe, rechaza la promesa con un mensaje de error.
• “Eliminar persona” → llama a la función deletePerson() que devuelve una promesa.
Si el teléfono existe, elimina la persona y muestra un mensaje de éxito.
Si no existe, rechaza la promesa con un mensaje de error.
Cada vez que se modifique el vector, se debe actualizar el listado en pantalla.
